What is Honeycomb?

Honeycomb is an open source MySQL storage engine that stores MySQL table data in an external data storage backend. With Honeycomb installed, MySQL is able to take advantage of the scalability, fault-tolerance, and tools of an underlying backend while preserving the familiar relational data model and SQL interface.

Honeycomb currently supports storing MySQL table data in HBase and an in-memory store. The HBase backend component provides access to the underlying data in offline map/reduce jobs without having to extract or duplicate the data. Existing applications using MySQL can be transitioned to use HBase with Honeycomb on a table-by-table basis with no changes to existing application code.

Support for additional storage engine backends can be implemented without knowledge of MySQL internals through high-level Java interfaces provided by Honeycomb.

How It Works

Honeycomb is organized into three distinct components: the storage engine plugin, proxy, and backend.

A high-level view of their interaction is shown in this figure:

The Honeycomb storage engine plugin is loaded by MySQL and is used to provide information from the MySQL optimizer to the storage engine proxy.

The proxy component is a Java library that is responsible for communicating the MySQL information to a storage engine backend.

A backend component implements Honeycomb's Java interfaces to define how supported operations should be performed by the storage implementation (for example, HBase) to process the SQL statements issued to MySQL.

For further details, take a look at the general architecture of Honeycomb.
